# Введение

recv — это **non-custodial крипто-платежный шлюз**. Клиенты платят вам on-chain, средства поступают **напрямую на ваш кошелек**, и recv никогда не касается денег. Вместо процента с каждой транзакции recv берет **фиксированную подписку** — комиссий с оборота нет.

## Что делает recv

- **Создает счета (invoices).** Вы формируете запрос на оплату в долларах для выбранной сети. recv пересчитывает цену в долларах в сумму к оплате on-chain, выбирает один из ваших кошельков как адрес назначения и возвращает ссылку на хостинговый checkout и платежный URI.
- **Следит за блокчейном.** Воркеры recv наблюдают за входящими переводами на ваши адреса назначения и сопоставляют их с открытыми счетами.
- **Подтверждает и уведомляет.** Когда найден соответствующий перевод, счет меняет статус (`paid`, `underpaid`, `overpaid`, `manual_review`), и recv отправляет подписанный вебхук на ваш endpoint.

```text
 Клиент              recv                         Вы
    │  платит on-chain  │                            │
    │ ─────────────────►│  назначение = ВАШ кошелек  │
    │                   │  (средства минуют recv)   │
    │                   │  сопоставляет перевод→счет │
    │                   │ ─── подписанный вебхук ───►│  обновить заказ
```

## Non-custodial модель

Вы подключаете кошельки для выплат по каждой сети в Консоли мерчанта. Каждый счет создается на *ваш* адрес, поэтому расчет идет напрямую клиент → мерчант. Роль recv — обнаружение, учет и уведомление, но не хранение средств. Нет шага вывода, нет баланса на стороне recv и нет процедуры withdrawal.

## Поддерживаемые сети

recv принимает платежи в **GRAM в TON, USDT в TON, TRON, Base и BSC**. См. [Поддерживаемые сети](/ru/docs/networks) для точного списка и примечаний по сетям.

## Два способа интеграции

1. **REST API (`/v1`).** Endpoint'ы с авторизацией по API-ключу для создания и управления счетами, плюс подписанные вебхуки об изменении статуса. Начните с [Быстрого старта](/ru/docs/quickstart).
2. **MCP-сервер (AI-агенты).** Сервер Model Context Protocol, который позволяет AI-агентам (Claude Desktop, Cursor и др.) создавать и проверять счета, верифицировать вебхуки и получать список сетей как инструменты. См. [MCP / AI-агенты](/ru/docs/mcp).

## С чего начать

Создайте аккаунт в [Консоли мерчанта](/app/auth), добавьте кошелек для выплат, затем выпустите API-ключ в Developer Portal. [Быстрый старт](/ru/docs/quickstart) проведет вас по всему пути с примерами curl.
