# Поддерживаемые сети

recv non-custodial: каждый счет оплачивается **напрямую на ваш собственный кошелек** в выбранной сети. Вы подключаете кошельки для выплат по каждой сети в Консоли мерчанта, и значение `payable_network`, передаваемое при [создании счета](/ru/docs/invoices), определяет, какой кошелек получит средства.

## Сети для оплаты

Это значения, принимаемые для `payable_network`:

| `payable_network` | Сеть | Wallet bucket |
|-------------------|------|---------------|
| `TON` | Нативный TON в The Open Network | TON |
| `TON_USDT` | USDT в The Open Network | TON |
| `TRON` | TRON (например, USDT TRC-20) | TRON |
| `BASE` | Base | EVM |
| `BSC` | BNB Smart Chain | EVM |

Значения регистронезависимы в запросе и нормализуются в верхний регистр.

### Wallet buckets

`BASE` и `BSC` делят единый **EVM wallet bucket** — один подключенный адрес `0x…` может быть назначением для обеих сетей. `TON` и `TON_USDT` делят TON bucket. `TRON` использует отдельный bucket. Чтобы создать счет в конкретной сети, у вас должен быть активный кошелек в соответствующем bucket.

## Сети кошельков

При подключении кошелька для выплат сеть — одна из `TON`, `TRON` или `EVM` (EVM-кошелек покрывает Base и BSC через общий bucket). Валидация адреса по сети:

| Сеть | Правило адреса |
|------|----------------|
| `TON` | Не менее 32 символов. |
| `TRON` | Начинается с `T`, не менее 20 символов. |
| `EVM` | Начинается с `0x`, ровно 42 символа. |

## Суммы и точность

`base_amount_usd` — десятичная строка в долларах с 2 знаками. recv пересчитывает ее в сумму к оплате on-chain `payable_amount`, возвращаемую с **6 знаками** для всех сетей.

## Комментарии / memo к платежу

Некоторые сети полагаются на memo для сопоставления перевода со счетом. Поле `payment_comment` объекта счета несет требуемый комментарий (пустое, если не нужно). Для **TON** `payment_uri` — это deep-link `ton://transfer/<address>?amount=<nanotons>&text=<comment>` с суммой в нанотонах; для **TON_USDT**, `TRON`, `BASE` и `BSC` `payment_uri` — это адрес назначения, а сопоставление идет по точной десятичной сумме. Показывайте точную сумму максимально заметно на checkout.

> recv поддерживает только перечисленные выше сети. Не предполагайте наличие других цепочек.
