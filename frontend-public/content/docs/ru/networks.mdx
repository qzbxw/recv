# Поддерживаемые сети

Reqst non-custodial: каждый счет оплачивается **напрямую на ваш собственный кошелек** в выбранной сети. Вы подключаете кошельки для выплат по каждой сети в Консоли мерчанта, и значение `payable_network`, передаваемое при [создании счета](/ru/docs/invoices), определяет, какой кошелек получит средства.

## Сети для оплаты

Это значения, принимаемые для `payable_network`:

| `payable_network` | Сеть | Wallet bucket |
|-------------------|------|---------------|
| `TON` | The Open Network | TON |
| `TRON` | TRON (например, USDT TRC-20) | TRON |
| `SOLANA` | Solana | Solana |
| `EVM` | Ethereum mainnet | EVM |
| `BASE` | Base | EVM |
| `ARBITRUM` | Arbitrum One | EVM |
| `BSC` | BNB Smart Chain | EVM |

Значения регистронезависимы в запросе и нормализуются в верхний регистр.

### Wallet buckets

Сети семейства EVM (`EVM`, `BASE`, `ARBITRUM`, `BSC`) делят единый **EVM wallet bucket** — один подключенный адрес `0x…` может быть назначением для любой из них. `TON`, `TRON` и `SOLANA` используют отдельные buckets. Чтобы создать счет в конкретной сети, у вас должен быть активный кошелек в соответствующем bucket.

## Сети кошельков

При подключении кошелька для выплат сеть — одна из `TON`, `TRON`, `SOLANA` или `EVM` (EVM-кошелек покрывает Base, Arbitrum и BSC через общий bucket). Валидация адреса по сети:

| Сеть | Правило адреса |
|------|----------------|
| `TON` | Не менее 32 символов. |
| `TRON` | Начинается с `T`, не менее 20 символов. |
| `EVM` | Начинается с `0x`, ровно 42 символа. |
| `SOLANA` | 32–44 символа, алфавит base58. |

## Суммы и точность

`base_amount_usd` — десятичная строка в долларах с 2 знаками. Reqst пересчитывает ее в сумму к оплате on-chain `payable_amount`, возвращаемую с **6 знаками** для всех сетей.

## Комментарии / memo к платежу

Некоторые сети полагаются на memo для сопоставления перевода со счетом. Поле `payment_comment` объекта счета несет требуемый комментарий (пустое, если не нужно). Для **TON** `payment_uri` — это deep-link `ton://transfer/<address>?amount=<nanotons>&text=<comment>` с суммой в нанотонах; для остальных сетей `payment_uri` — это адрес назначения. Когда сеть требует комментарий, клиент должен его указать, иначе перевод может не сопоставиться автоматически — отображайте его заметно на checkout.

> Reqst поддерживает только перечисленные выше сети. Не предполагайте наличие других цепочек.
