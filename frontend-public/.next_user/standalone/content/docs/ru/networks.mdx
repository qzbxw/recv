# Поддерживаемые сети

recv non-custodial: каждый счет оплачивается **напрямую на ваш собственный кошелек** в выбранной сети. Вы подключаете кошельки для выплат по каждой сети в Консоли мерчанта, и значение `payable_network`, передаваемое при [создании счета](/ru/docs/invoices), определяет, какой кошелек получит средства.

## Сети для оплаты

Это значения, принимаемые для `payable_network`:

| `payable_network` | Активы | Wallet bucket |
|-------------------|--------|---------------|
| `TON` | TON | TON |
| `TON_USDT` | USDT | TON |
| `TRON` | USDT | TRON |
| `SOLANA` | SOL, USDT, USDC | SOLANA |
| `BASE` | USDT, USDC | EVM |
| `ARBITRUM` | USDT, USDC | EVM |
| `BSC` | BNB, USDT | EVM |

Значения регистронезависимы в запросе и нормализуются в верхний регистр.

### Wallet buckets

`BASE`, `ARBITRUM` и `BSC` делят единый **EVM wallet bucket**. `TON` и `TON_USDT` делят TON bucket. `TRON` и `SOLANA` используют отдельные buckets.

## Сети кошельков

При подключении кошелька для выплат сеть — одна из `TON`, `TRON`, `SOLANA` или `EVM`. Валидация адреса по сети:

| Сеть | Правило адреса |
|------|----------------|
| `TON` | Не менее 32 символов. |
| `TRON` | Начинается с `T`, не менее 20 символов. |
| `SOLANA` | Solana base58-style адрес, 32-44 символа. |
| `EVM` | Начинается с `0x`, ровно 42 символа. |

## Суммы и точность

`base_amount_usd` — десятичная строка в долларах. Stablecoins считаются 1:1 к USD. Нативные TON, SOL и BNB пересчитываются при создании счета; счета с volatile native asset получают более короткое окно оплаты.

## Комментарии / memo к платежу

Счет может содержать до двух `payment_options`. Для нативного TON option содержит обязательный memo в `payment_comment`; для stablecoins и других native options сопоставление идет по точной сумме, сети, активу и адресу назначения.

> recv поддерживает только перечисленные выше сети. Не предполагайте наличие других цепочек.
