Протокол

Возможности API и Архитектура

recv работает как прозрачный программный слой (middleware). Транзакции идут напрямую от клиента к вам, минуя промежуточные счета. Мы лишь автоматизируем мониторинг через сеть высокопроизводительных нод.

Единый мультичейн шлюз

Хватит поддерживать кастомные RPC ноды и хрупкие слушатели блокчейнов. Одна стандартная интеграция REST API покрывает нативный TON, USDT в TON, TRON, Base и BSC.

Криптографические вебхуки

Изменения статуса могут вызывать POST-вебхуки на ваш сервер. HMAC-SHA256 подпись и timestamp позволяют backend проверить источник и применить защиту от повторов.

Нативная идемпотентность

Никогда не начисляйте баланс пользователю дважды за одну и ту же блокчейн-транзакцию. Наш основной протокол обеспечивает строгую идемпотентность, защищая вашу базу данных от дублирующейся обработки во время перегрузки сети или микро-форков.

Testnet и Sandbox режим

Разрабатывайте и тестируйте вашу интеграцию безопасно, используя нашу среду Sandbox. Генерируйте моковые счета и симулируйте подтверждения в блокчейне без риска реальным капиталом.

FAQ

Часто задаваемые вопросы

Когда клиент инициирует перевод, наши вотчеры мониторят соответствующий блокчейн. Как только транзакция достигает нужных подтверждений, recv отправляет JSON payload через HTTP POST на ваш endpoint с ID счета, суммой и TX hash.

recv использует резервируемый RPC-доступ для поддерживаемых сетей. Если публичная нода падает, мы переключаемся на резервных провайдеров. Во время перегрузки вотчеры сохраняют состояние и доставляют вебхук после финализации платежа.

Идемпотентность гарантирует, что независимо от того, сколько раз повторяется конкретный API запрос или вебхук (из-за сбоев сети или ошибок клиента), конечный результат остается неизменным. В крипте это жизненно важно для предотвращения 'двойных трат' или двойного предоставления пользователю премиум-подписки за одну блокчейн-транзакцию.

API recv использует протокол умного сопоставления (smart-matching) вместо генерации уникальных адресов кошельков для каждого счета (что дорого и сложно сводить/свипать). Мы используем математически уникальные десятичные окончания (например, 100.000123 USDT) или нативные memo сети (на TON), чтобы мгновенно связать одну входящую транзакцию на вашем мастер-кошельке с конкретным счетом пользователя.

В настоящее время мы предоставляем подробно задокументированный RESTful API, не зависящий от языка, который может использоваться любым современным HTTP-клиентом (Axios, Fetch, Requests). Официальные SDK для TypeScript/Node.js и Go находятся в активной разработке.

Каждый вебхук включает заголовок 'X-recv-Signature'. Вы должны использовать ваш секретный ключ вебхука (Webhook Key), чтобы вычислить хэш HMAC-SHA256 сырого тела запроса (raw request body). Если ваш вычисленный хэш совпадает с нашим заголовком, payload является подлинным и не был подделан.

Мы не устанавливаем жесткую блокировку вашего API, если у вас произошел всплеск (spike). Вы получите автоматическое уведомление по email и Telegram при приближении к лимиту (80% и 100%). Перерасход может быть оплачен по номинальной ставке, или вы можете бесшовно перейти на тариф Business.
