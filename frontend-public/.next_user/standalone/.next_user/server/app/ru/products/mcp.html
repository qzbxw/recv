Как это работает

Платежи для эпохи агентов

Добавьте MCP-сервер recv в любой MCP-совместимый хост. Агент сразу получает 12 инструментов для онбординга, биллинга и проверки платежей — без дашборда и ручной настройки.

12 инструментов агента

bootstrap_agent_workspace

Саморегистрация воркспейса, получение access token

create_subscription_checkout

Покупка плана — merchant, developer или business

get_checkout_invoice

Поллинг оплаты подписки до подтверждения

get_account

Чтение воркспейса, плана и баланса

create_invoice

Создание платежного инвойса на любой сети

get_invoice

Получение статуса существующего инвойса

list_invoices

Пагинация истории инвойсов

simulate_payment

Тестовый платеж в sandbox-режиме

create_api_key

Выдача скопированного API-ключа после активации плана

create_webhook_endpoint

Регистрация HTTPS-эндпойнта для событий

verify_webhook

Проверка HMAC-SHA256 подписи локально

list_supported_networks

Список доступных блокчейн-сетей

Агентное отличие

Почему продукт нужен до выбора тарифа

Без recv

Агенты не могут регистрироваться, купить планы или получать API-ключи без участия человека.

С recv

bootstrap_agent_workspace создаёт воркспейс и возвращает access token за один вызов.

Без recv

LLM не могут нативно создавать и проверять платежные инвойсы.

С recv

create_invoice и get_invoice — полноценные MCP-инструменты, которые агент вызывает напрямую.

Без recv

Проверка подписи вебхука требует ручного кода на бекенде и работы с секретами.

С recv

verify_webhook работает локально в MCP-процессе — ни сетевых запросов, ни утечки секретов.

{
  "mcpServers": {
    "recv": {
      "command": "npx",
      "args": ["-y", "recv-mcp"],
      "env": {
        "RECV_API_KEY": "live_...",
        "RECV_ACCESS_TOKEN": "your_token"
      }
    }
  }
}
01

Онбординг агента без участия человека

AI-агент может полностью зарегистрироваться в recv без каких-либо действий человека: вызывает bootstrap_agent_workspace, получает access token, покупает Developer или Business-подписку через create_subscription_checkout, поллит get_checkout_invoice до оплаты, затем выдаёт себе API-ключ. Весь флоу — последовательность MCP-вызовов.

02

Любой MCP-совместимый рантайм

MCP-сервер recv работает через stdio и поддерживает Claude Desktop, Claude Code, Cursor, Cline, Continue и любой другой хост, реализующий спецификацию Model Context Protocol. Добавьте его в mcp.json с RECV_API_KEY и RECV_ACCESS_TOKEN — и ваш агент сразу получает платежные суперсилы.

03

Безопасность по дизайну

Проверка вебхуков происходит внутри MCP-процесса — разрытое тело запроса и секрет не покидают локальный рантайм. API-ключи скопированы на invoices:read и invoices:write. Агент видит только нужное, а каждое платежное событие криптографически подписывается перед доставкой.

AI-NATIVE

Дайте вашему агенту возможность зарабатывать.

Первая крипто-инфраструктура для AI-агентов. recv MCP позволяет любому Claude, GPT или кастомному LLM создать воркспейс, подписаться на план и начать принимать платежи без участия человека.

Документация MCP→
Интеграция recv MCP (Модель Контекстного Протокола) позволяет AI-агентам и LLM-приложениям автономно управлять полным циклом криптоплатежей. Агенты в Claude Desktop, Claude Code, Cursor или любом другом MCP-совместимом хосте могут вызывать bootstrap_agent_workspace для саморегистрации, покупать подписку, генерировать API-ключи, создавать инвойсы, регистрировать webhook-эндпойнты и проверять подписи вебхуков локально. recv — первая платежная инфраструктура, нативно разработанная для агентной эпохи автономного ПО.
