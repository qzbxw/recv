API и инфраструктура для масштабирования

recv API дает разработчикам создание инвойсов, мониторинг оплат и подписанные callback-уведомления, оставляя тарифные решения вне интеграционного сценария.

Webhooks

Статусы detected, underpaid, paid и expired приходят на ваш backend.

HMAC/idempotency

Проверяйте подписи вебхуков и используйте idempotency keys против дублей.

Error handling

Предсказуемые HTTP-коды для валидации, авторизации, лимитов и сетевых ошибок.

Rate limits

Понятные лимиты запросов с переходом на business-объемы при росте нагрузки.

01

Повторная доставка вебхуков

Движок использует очередь повторных доставок. Если сервер недоступен, попытки повторяются в пределах бюджета активного тарифа. Каждый payload подписывается HMAC, а backend должен проверять подпись и timestamp.

02

Абстракция блокчейн-сетей

Интеграция TON, TRON и EVM обычно требует разных библиотек и логики. recv дает единую схему для всех сетей. Создавайте инвойс для TRC-20 USDT так же просто, как для нативного TON или USDT в TON.

03

Высокопроизводительные вотчеры

recv наблюдает поддерживаемые сети и обновляет статусы после обнаружения и требуемых подтверждений. Время зависит от конкретной сети и доступности её RPC-инфраструктуры.
